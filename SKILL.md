# Tab Out — 标签页管理助手

## 这个 skill 做什么

帮用户在 1 分钟内安装 **Tab Out** Chrome 扩展，解决「标签页太多管不过来」的问题。

---

## 解决谁的问题

### 你是否有过这样的体验？

> 早上打开电脑，打开 3-4 个 YouTube 视频了解前沿技术，又开 5-6 个浏览器 tab 调研竞品方案，同时还有 CRM、OA、文档协作等公司内部系统需要保持开着。
>
> 到下午，浏览器已经开了 **20+ 个标签页**。你想找上午看的那个视频？翻半天找不到。想关掉调研完的那些页面？一个个关太麻烦。
>
> 最后的结果是：标签页越来越多，但每个都不想关，浏览器越来越卡，工作效率越来越低。

### Tab Out 解决的 3 个核心问题

| 痛点 | 解决方案 |
|------|----------|
| **标签页太多，找不到想要的** | 按域名自动分组 + 全局搜索，⌘K 一搜就到 |
| **想关但不敢关，怕忘了** | 收藏功能，存到右侧清单，随时可以重新打开 |
| **重复打开的页面浪费资源** | 自动检测重复，一键保留 1 个或全部关闭 |

### 适用人群

- **产品经理 / 设计师** — 同时打开多个竞品页面做对比
- **开发者** — 多个 localhost 项目同时开发，需要快速切换
- **研究员 / 内容创作者** — 大量资料页面需要分类管理
- **任何每天打开 10+ 个标签页的人**

## 安装引导流程

### 第 1 步：介绍产品

告诉用户他们即将获得什么：

> **Tab Out** 把你的新标签页变成一个标签页管理仪表盘。
>
> - 所有标签页按域名自动分组，一目了然
> - 自定义分类名称（比如把 crm.xxx.com 改成「CRM 系统」）
> - 搜索所有标签页，⌘K 快捷键直达
> - 收藏标签页到稍后阅读清单，随时可以重新打开
> - 重复标签页自动检测，一键去重
> - 治愈系配色，看着就放松
> - 100% 本地，数据不上云
>
> 安装只需 1 分钟，无需任何账号或服务器。

### 第 2 步：克隆仓库

```bash
git clone https://github.com/zarazhangrui/tab-out.git
cd tab-out
```

### 第 3 步：安装扩展

**首先**，打印扩展文件夹路径并复制到剪贴板：
```bash
# macOS
echo "扩展文件夹路径: $(cd extension && pwd)"
cd extension && pwd | pbcopy && echo "✅ 路径已复制到剪贴板"

# 打开文件浏览器（备用方案）
open extension/
```

**然后**，引导用户操作：

> 我已经把扩展文件夹路径复制到你的剪贴板了。请按以下步骤操作：
>
> 1. 打开 Chrome，地址栏输入 `chrome://extensions` 回车
> 2. 右上角开启 **开发者模式** 开关
> 3. 点击左上角 **「加载已解压的扩展程序」**
> 4. 在文件选择器中按 **⌘+Shift+G**（Mac）或 **Ctrl+L**（Windows），粘贴路径后回车
> 5. 点击「选择」完成安装
>
> 打开一个新标签页，就能看到 Tab Out 仪表盘了。

### 第 4 步：功能导览

安装完成后，告诉用户怎么用：

> **基本操作：**
> 1. 所有标签页按域名分组显示，每个分组有中文分类名
> 2. 点击标签页标题 → 直接跳转到那个标签页
> 3. 点击 ✕ → 关闭单个标签页（带音效动画）
> 4. 点击「关闭全部 N 个标签页」→ 关闭整个分组
>
> **进阶功能：**
> 5. 点击 ✏️ 编辑按钮 → 给域名组起自定义中文名
> 6. 按 ⌘K → 打开搜索框，搜索所有标签页
> 7. 点击顶部标签 → 按分类筛选
> 8. 点击收藏按钮 → 标签页保存到右侧清单
> 9. 收藏的标签页可以一键恢复重新打开
> 10. 重复标签页会显示 ×N 标记，底部有「关闭 N 个重复」按钮

## 技术信息

- **纯 Chrome 扩展**，无需 Node.js、npm 或任何构建工具
- **数据存储**：`chrome.storage.local`（跨会话持久化，数据仅存本地）
- **音效**：Web Audio API 合成，无需音频文件
- **动画**：CSS 过渡 + JS 粒子效果
- **更新方式**：`git pull` 后在 `chrome://extensions` 点击刷新按钮
